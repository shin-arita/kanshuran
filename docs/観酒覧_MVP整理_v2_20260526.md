# 観酒覧 MVP整理 v2
作成日: 2026-05-26

## 結論

観酒覧MVPの境界と主要UXを整理した。

観酒覧は、

- 黒子
- 店舗主役
- 店運用支援
- できるだけ軽く
- 設定・入力を増やさない
- 雑でも回る

を思想の中心とする。

---

# 思想

観酒覧は店舗支援基盤である。

主敵:

- 紙
- 記憶
- ボトル探索時間

営業メッセージ:

- ボトル探しの時間、なくしませんか？

観酒覧価値:

- 探索時間削減
- 記憶補助
- 接客品質向上
- 来店前準備
- 再来店促進

# MVP目的

店員が来店客のキープを、

- 速く
- 迷わず
- 記憶依存なく

提供できること。

# ボトル / キープ

## 定義

- 物理ボトル = bottle
- 預かり状態 = keep
- キープ情報 = ボトル写真 + 銘柄

## ボトルモデル

- ボトル : 人 = 1:N
- 個人/共有を分けない
- グループキープあり

## 状態

保管中  
↓  
終了  
↓  
履歴

- 削除なし

## 終了

期限切れ  
≠  
自動終了

導線:

保管中  
↓  
期限切れ（通知）  
↓  
店判断  
↓  
終了  
↓  
履歴

期限は、

- 管理目安
- 記憶補助

であり、強制ルールではない。

## 再キープ

履歴  
↓  
コピー新規

# 通知

## 基本思想

通知は、

- 厳格管理
- タスク化

ではない。

やさしい記憶補助。

## 客向け通知

チャネル:

- LINE Push

名義:

- 店舗名

対象:

- 連携者全員

デフォルト:

- 14日前
- 7日前
- 前日
- 期限切れ

店設定:

- 任意変更可
- OFF可

## 店向け通知

チャネル:

- LINE
- PWA

役割:

LINE = 気づき  
PWA = 管理

送り先:

- 店設定
- デフォルト = 店員全員

状態:

未対応  
↓  
対応済  
（再対応不可）

通知は:

- 気づく
- 一度対応する
- 二重対応しない

だけ。

対応メモ・解除等は不要。
