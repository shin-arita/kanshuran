# 観酒覧 LineCustomer・LINE連携ドメイン設計 v1
作成日: 2026-05-27

## 結論

観酒覧MVPにおけるLineCustomer・LINE連携ドメインを整理した。

観酒覧はLINE文化を前提とする。

ただし、

- 顧客統合管理
- CRM化
- 全店舗共通顧客マスタ

は採用しない。

観酒覧のLineCustomerは、

「店舗文脈のLINE連携済み客」

である。

---

# ドメインモデル

```text
Store 1 ── 0..N LineCustomer

Bottle 1 ── 0..N BottleLineCustomer
LineCustomer 1 ── 0..N BottleLineCustomer
```

つまり、

```text
Bottle N ── N LineCustomer
```

グループキープは MVP IN。

Bottle側:

```text
Bottle
├─ owner_name nullable
├─ owner_note nullable
└─ (LINE連携時は BottleLineCustomer で紐付け)
```

GuestCustomer は採用しない。

---

# LineCustomer（LINE連携済み客）

LineCustomer は、

観酒覧に接続した客
=
LINE連携済み客

である。

LineCustomer は店舗単位で存在する。

責務:

- マイキープ利用主体
- LINE連携主体
- 店舗との関係主体

LineCustomer は、

「店が認識する客」

ではなく、

「LINE経由で観酒覧に接続した客」

である。

---

# LINE連携識別

LINE Login による識別情報は LineCustomer が持つ。

LineCustomer は LINE user id を保持する。

LINE Login識別と LineCustomer は 1:1 とする。

---

# 店舗文脈LineCustomer

観酒覧は、

全店舗共通LineCustomer

を採用しない。

理由:

店ごとに客像・呼称・関係性が異なるため。

同じLINEユーザが、
複数店舗のLineCustomerになってよい。

例:

```text
LINE_ID = xxx（実は同じ人物）

A店:
愉快なおっちゃん山田くん

B店:
クールな川崎くん
```

これは許容する。

観酒覧は、

LINE user id
≠
LineCustomer identity

と考える。

LineCustomer は、

店舗文脈の客

である。

---

# LINE未連携キープ

LINE未連携の持ち主情報は、
LineCustomer として管理しない。

Bottle側で吸収する。

理由:

- 店運用を止めない
- 設定より先に成功体験
- 雑でも回る
- CRM化しない

ため。

LINE未連携 Bottle でできること:

```text
店員が名前でキープを検索できる
店員が写真・銘柄・保管場所・残量を見られる
店員が来店時に探せる
```

LINE未連携 Bottle でできないこと:

```text
客がマイキープで見られない
LINE通知を送れない
```

フロー:

```text
店員キープ登録
↓
Bottle作成
  owner_name
  owner_note
↓
店側管理可能
↓
店QR
↓
LINE Login
↓
LineCustomer作成
↓
BottleLineCustomer 紐付け
↓
マイキープ利用
```

---

# Bottle側持ち主情報

Bottle は、
LINE連携前でも存在できる。

```text
owner_name nullable
owner_note nullable
```

とする。

owner_name は nullable。

理由:

現場では、

- 名前未確認
- 団体客
- 仮登録

があり得るため。

例:

```text
owner_name = null
owner_note = カウンター右の黒霧
(BottleLineCustomer なし)
```

これは許容する。

---

# 非採用（MVP OUT）

以下は採用しない。

- GuestCustomer
- Customer という曖昧名称
- 仮顧客モデル
- 全店舗共通顧客
- CRM統合
- 顧客統合ID

---

# 設計思想

観酒覧のLineCustomer・LINE連携設計は、

- LINE文化
- 店舗文脈
- 黒子思想
- 雑でも回る

を優先する。

観酒覧は、

顧客統合管理システムではない。

店と客の関係を、
ゆるく支援する黒子である。
