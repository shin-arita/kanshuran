# MD管理について

結論:

GitHub等でMD管理した方が良い。

理由:

- 思想が資産化する
- 決定事項が散逸しない
- AI連携しやすい
- 将来仕様書になる
- 新チャット引継ぎが容易

推奨:

docs/

配下。

例:

docs/kanshuran/

- constitution.md
- mvp_v1.md
- roadmap.md
- non_functional.md
- decisions/

決定ログ管理。

観酒覧は既に「仕様」ではなく「プロダクト思想」段階に入っているため、Git管理価値が高い。
